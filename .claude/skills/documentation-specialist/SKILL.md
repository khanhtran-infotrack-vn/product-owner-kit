# Documentation Specialist

## Core Capabilities
- Generate product documentation and user guides
- Create API documentation with examples
- Maintain Architecture Decision Records (ADRs)
- Update wiki/Confluence pages
- Generate FAQs from support tickets
- Version control for requirements
- Create onboarding documentation

## Output Examples

### Feature Documentation

```markdown
# [Feature Name]

## Overview
[Brief description of the feature and its purpose]

## User Guide

### Getting Started
1. [Step 1]
2. [Step 2]
3. [Step 3]

### Best Practices
- [Practice 1]
- [Practice 2]
- [Practice 3]

### Troubleshooting
**Problem:** [Common issue]
- **Cause:** [Why it happens]
- **Solution:** [How to fix]

## Technical Details
- Supported formats: [List]
- Limitations: [Constraints]
- Performance: [Metrics]
```

### Architecture Decision Record (ADR)

```markdown
# ADR-[Number]: [Title]

## Status
[Accepted | Rejected | Deprecated | Superseded by ADR-XXX]

## Context
[Describe the problem and alternative options considered]

## Decision
[State the decision made]

## Rationale
- [Reason 1]
- [Reason 2]
- [Reason 3]

## Consequences

**Positive:**
- [Benefit 1]
- [Benefit 2]

**Negative:**
- [Trade-off 1]
- [Trade-off 2]

## Implementation Notes
- [Technical detail 1]
- [Technical detail 2]
```

### Release Notes

```markdown
# Release v[X.Y.Z] - [Date]

## New Features
- **[Feature Name]**: [Description of what users can now do and the benefit]

## Improvements
- **[Change]**: [Impact on user experience or performance]

## Bug Fixes
- **[Issue]**: [What was fixed and the improvement users will see]

## Breaking Changes
- **[Change]**: [What changed and what users need to do]

## Known Issues
- [Issue description and workaround if available]
```

### FAQ Generation

```markdown
# Frequently Asked Questions

## [Category Name]

**Q: [Common question]**
A: [Clear, concise answer with examples if needed]

**Q: [Another question]**
A: [Answer with links to related documentation]
```
